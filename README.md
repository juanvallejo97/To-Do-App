# To-Do-App

A task management application built with FlutterFlow for CSC305, featuring user authentication, profile management, task CRUD operations, and inspirational quotes.

This project follows agile sprint-based development methodology with comprehensive project management through GitHub Projects.

## 📊 Project Management

### GitHub Project Board Structure

The project uses a Kanban-style board with four columns:
1. **Product Backlog** - All unscheduled user stories
2. **Sprint Backlog** - Work planned for current sprint
3. **In Progress** - Active development tasks
4. **Done** - Completed and verified work

### Labels

- **User Stories**: US1, US2, US3, US4, US5, US6
- **Sprints**: Sprint 1, Sprint 2
- **Types**: PBI (Product Backlog Item), Task
- **Assignment**: [Your Name]

### Workflow

1. **Sprint Planning**: Move PBIs from Product Backlog → Sprint Backlog
2. **Development**: Move tasks Sprint Backlog → In Progress → Done
3. **Sprint Review**: Verify all items in Done column
4. **Retrospective**: Review and plan improvements

For complete workflow details, see [GITHUB_PROJECT_SETUP.md](GITHUB_PROJECT_SETUP.md).

## 👥 Contributing

This is an academic project for CSC305. Contributions follow the agile sprint workflow:

1. Review current sprint in [SPRINT_2.md](SPRINT_2.md)
2. Pick a task from Sprint Backlog
3. Move task to In Progress on project board
4. Create feature branch: `git checkout -b feature/task-name`
5. Implement changes following acceptance criteria
6. Test thoroughly
7. Submit pull request
8. Move task to Done after review
